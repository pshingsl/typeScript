1) 클래스
 - 클래스는 '객체의 설계도'
 - 현실 세계의 구조와 행동을 코드로 추상화 한것
 - 추상적인 이유: 구체적이지 못해서
 - 클래스의 구성 요소
   - 클래스에서는 같은 종류의 객체들이 공통으로 가지는 속성(attribute)과 메서드(method)정의
    속성:클래스에서 사용하는 변수
     예시) 붕어빵이 있음 팥, 슈크림 등등 = 변수로 생각하자 
     변수의 값을 어떤 걸 가지는걸 따라 달라짐
    메서드:클래스에서 가지는 함수-> 동작
   - **속성**은 **객체의 성질(변수 값)**
